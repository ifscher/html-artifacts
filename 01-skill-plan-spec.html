Planejamento de skill · html-artifact

Skill html-artifact

Gera artefatos HTML auto-contidos no design system estabelecido, salva no projeto atual e mantém o catálogo atualizado automaticamente.

Resumo

Você descreve o que quer. A skill escolhe o template mais próximo dos demos de referência, gera um HTML com os tokens de design corretos, salva em artifacts/ do projeto ativo e registra um card no artifacts/index.html. Se a pasta não existir, cria junto com o índice.

01Inputs▶

A skill recebe dois parâmetros:

ParâmetroObrigatórioDescrição
descriçãosimO que o artefato deve mostrar ou fazer. Ex: "plano de implementação do módulo de auth"
tiponãoHint do tipo: spec, report, explorer, editor, deck. Se omitido, a skill infere pelo conteúdo.
02Contexto carregado pela skill▶

A skill injeta no contexto antes de gerar:

  • Os tokens CSS do design system (--ivory, --clay, --serif, etc.)
  • Os padrões de layout mais usados (card, section, grid, thumb)
  • O demo de referência mais próximo do tipo solicitado, como exemplo concreto
  • Regra: HTML auto-contido, zero dependências externas, CSS e JS inline
03Estrutura de pastas gerada▶
projeto/
  src/
  ...
  artifacts/ ← criado na primeira vez
    index.html ← catálogo do projeto
    01-nome-do-artefato.html
    02-outro-artefato.html
    ...

O index.html do projeto referencia os artefatos locais + link para ifscher.github.io/html-artifacts/thariq/ como referência de design.

04Comportamento do índice▶

A skill atualiza o artifacts/index.html automaticamente a cada novo artefato:

  • Adiciona um card com título, descrição e link para o arquivo
  • Incrementa a numeração (NN-) baseado nos arquivos existentes
  • Seção "Referência" no rodapé linka para o catálogo central no GitHub Pages
  • Nunca reescreve o índice do zero — sempre faz append do card novo
05Edge cases▶
Primeiro artefato
Pasta não existe
Cria artifacts/ e gera o index.html do zero com o novo card e a seção de referência.
Índice sem referência
Index antigo sem seção ref
Adiciona a seção de referência ao final antes de registrar o novo card.
Tipo ambíguo
Skill não consegue inferir
Pergunta ao usuário com opções: spec / report / explorer / editor / deck.
Nome de arquivo
Conflito de numeração
Lê os arquivos existentes em artifacts/ e usa o próximo número disponível.
06Decisões tomadas▶
Decidido
Sem .gitignore automático
Artefatos ficam versionados por padrão. Repos são majoritariamente privados — se formatar o PC ou perder a máquina, os artefatos estão no histórico. O usuário adiciona ao .gitignore manualmente se quiser.
Decidido
Thumbnail SVG gerado por tipo
Cada card no índice recebe um SVG simples baseado no tipo do artefato (spec = documento, editor = dois painéis, report = gráfico, deck = slides). Nada genérico.
Decidido
Funciona fora de projeto git
Sem restrição. A skill cria a pasta artifacts/, gera o HTML e atualiza o índice independentemente de haver um repositório git.
Decidido
Artefatos commitados no repo
Versionados junto com o código por padrão. São documentação permanente do projeto, não descartáveis.
