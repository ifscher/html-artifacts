04

Código principal

As duas partes com maior risco de serem feitas errado: a migration (soft deletes, cursores de leitura) e a mutation otimista (reconciliação de temp-id).

packages/db/migrations/0042_comments.sql
create table comments (
  id          uuid primary key default gen_random_uuid(),
  task_id     uuid not null references tasks(id),
  parent_id   uuid references comments(id),   -- apenas um nível,
                                                -- aplicado na API
  author_id   uuid not null references users(id),
  body        text not null,
  created_at  timestamptz not null default now(),
  deleted_at  timestamptz                       -- soft delete
);

create table comment_reads (
  task_id    uuid not null references tasks(id),
  user_id    uuid not null references users(id),
  read_up_to timestamptz not null,
  primary key (task_id, user_id)
);

create index comments_task_created
  on comments (task_id, created_at);
apps/web/hooks/useAddComment.ts
export function useAddComment(taskId: string) {
  const qc = useQueryClient();
  return trpc.comments.create.useMutation({
    onMutate: async (input) => {
      const temp = { ...input, id: `temp-${nanoid()}`,
                     createdAt: new Date(), pending: true };
      qc.setQueryData(key(taskId), (prev) =>
        [...(prev ?? []), temp]);
      return { tempId: temp.id };
    },
    onSuccess: (row, _v, ctx) => {
      // reconciliar id temp → id real para que o
      // append em tempo real não duplique
      qc.setQueryData(key(taskId), (prev) =>
        prev.map((c) => c.id === ctx.tempId ? row : c));
    },
    onError: (_e, _v, ctx) => {
      qc.setQueryData(key(taskId), (prev) =>
        prev.filter((c) => c.id !== ctx.tempId));
    },
  });
}
05

Riscos & mitigações

Risco
Sev
Mitigação
Duplicata em tempo real: o append via socket entra em race com a resposta HTTP e a reconciliação do temp-id.
ALTO
Deduplicar pelo id atribuído pelo servidor no updater do cache; o payload do socket carrega o id real, e linhas temp são filtradas na reconciliação.
Contagens de não lidos ficam desatualizadas quando o usuário lê o thread em outro dispositivo.
MED
Transmitir upserts de comment_reads no mesmo canal; o cliente trata seu próprio cursor como max(local, remoto).
Falsos positivos na detecção de menção em markdown colado (@media, @2x).
BAIXO
Resolver menções apenas contra membros do workspace, no momento da escrita, e armazenar os user ids resolvidos — nunca re-parsear na leitura.
06

Questões em aberto

Permitimos edição ou apenas excluir e repostar?
Edição requer uma coluna edited_at e uma affordance de "editado". Excluir e repostar é mais simples, mas perde a âncora de resposta. Tendência para excluir apenas no v1.
Decidir com · design, antes da fatia 2
Cadência do digest por e-mail quando o usuário está com o app fechado
Imediato por menção vai ser muito ruidoso. Proposta: agrupar em uma janela de 15 minutos, colapsar em um e-mail por tarefa e respeitar o horário de silêncio da tabela de configurações existente.
