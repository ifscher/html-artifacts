TL;DR

Os envios de notificação estavam acontecendo inline no caminho da requisição. Sob carga, adicionavam 200–800 ms à latência das mutations e descartavam e-mails silenciosamente quando o pool SMTP era esgotado. Este PR move a entrega para a fila pg-boss existente, para que a API retorne imediatamente e envios com falha sejam refeitos com backoff.

Por que

Quando adicionamos @menções nos comentários no trimestre passado, cada menção passou a disparar até três envios (in-app, e-mail, Slack) dentro da mesma transaction que salvava o comentário. Isso era aceitável no lançamento. Não é mais aceitável agora que uma única atualização de tarefa pode fazer fan-out para quarenta observadores.

Antes
  • Envios rodam inline no handler da mutation
  • Timeout SMTP = erro 500 no comentário
  • Sem retries — e-mail descartado é perdido
  • p99 em comments.create: 1,4 s
Depois
  • Handler enfileira um job por destinatário e retorna
  • Worker refaz 3× com backoff exponencial
  • Tabela dead-letter para inspeção após esgotamento
  • p99 em comments.create: 180 ms (staging)

Arquivo por arquivo

Ordenado para leitura, não alfabeticamente. Comece pelo worker — é o que há de novo — depois o ponto de enqueue, depois o restante.

packages/notify/src/worker.ts novo +126

O coração do PR. Um subscriber pg-boss que consome jobs notify.deliver, resolve as preferências de canal do usuário e chama o adapter correto. Os retries são configurados por canal — e-mail recebe três tentativas, Slack recebe uma porque sua API já é idempotente do nosso lado.

boss.work('notify.deliver', { batchSize: 20 }, async (jobs) => {
  for (const job of jobs) {
    const { userId, event, channel } = job.data;
    const prefs = await getPrefs(userId);
    if (!prefs[channel]) return;          // usuário silenciou este canal

    try {
      await adapters[channel].send(userId, event);
    } catch (err) {
      if (job.retryCount >= MAX_RETRY[channel]) {
        await deadLetter(job, err);         // não lança — ack & estaciona
        return;
      }
      throw err;                            // pg-boss reagendará
    }
  }
});
